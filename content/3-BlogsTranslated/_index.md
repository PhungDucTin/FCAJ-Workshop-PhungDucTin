### [Blog 1 - Internal Social Network Project "Mini Social Network" – Applying 3-Tier Architecture on AWS](3.1-Blog1/)

This blog shares the design and deployment process of an internal social network project named "Mini Social Network" on the AWS Cloud platform. This project is solidly built based on the standard 3-Tier Architecture model to ensure maximum data isolation and security. The system architecture is transparently separated into two main flows: the Data Plane, responsible for handling user access traffic, and the Control Plane, which handles operations, CI/CD automation, and the underlying monitoring system. This blog post will analyze the system's operating mechanism in detail while opening up a discussion space on important infrastructure upgrade solutions (Phase 2) such as integrating CloudFront, using VPC Endpoints, and migrating CI/CD to Cloud-Native.

### [Blog 2 - A Diary of Bringing an App to the Cloud – From a $35 Lesson to a Cost-Optimized Architecture](3.2-Blog2/)

This blog is the next part of the article series on the "Mini Social Network" project, where the development team shares their practical journey of deploying the application to the AWS environment. This article acts as a diary summarizing the mistakes made from the initial mindset of bundling everything into a single virtual private server (VPS), up to the turning point of transforming and optimizing towards a Cloud-Native architecture. You will discover in detail how each department solved their specialized problems: from the infrastructure team cutting NAT Gateway costs (from $35/month down to $0) and the Frontend team handling SPA routing errors by integrating CloudFront, to incorporating proactive monitoring and setting up DevSecOps right from the design phase. Through this, the blog provides a realistic perspective on balancing performance, automation capabilities, and cost optimization on the Cloud.
