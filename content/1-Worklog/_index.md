This page summarizes the entire 12-week practical journey in the **First Cloud AI Journey** program, focusing on the AWS platform, architecture, and application security testing. The content is presented in a logical progression: from getting acquainted with **basic AWS infrastructure** and establishing a multi-layer defense system **(Defense-in-depth)** to deploying the complete v3 Architecture capable of handling high loads and DDoS protection, while actively contributing value to the technical community.

Key milestones of this journey:

* **Week 1:** [Familiarization with AWS Cloud, IAM, VPC, EC2, and performing a system audit](1.1-week1/)

* **Week 2:** [Practicing AWS labs, researching AWS WAF, and proposing Managed Rules](1.2-week2/)

* **Week 3:** [Enhancing security with IAM Permission Boundaries, KMS, Secrets Manager, and AWS WAF](1.3-week3/)

* **Week 4:** [Exploring centralized networking with VPC, Transit Gateway, Route 53, and WAF protecting ALB](1.4-week4/)

* **Week 5:** [Learning HA for Windows/SQL Server and performing DAST with OWASP ZAP](1.5-week5/)

* **Week 6:** [Applying DevSecOps, CI/CD, serverless, and configuration management on AWS](1.6-week6/)

* **Week 7:** [Deploying Parameter Store, AWS WAF on ALB, and writing project security reports](1.7-week7/)

* **Week 8:** [Performing Authenticated DAST, testing BAC/IDOR, and preparing for live demo](1.8-week8/)

* **Week 9:** [Leading content creation, synthesizing practical experiences, and publishing Technical Blog #2](1.9-week9/)

* **Week 10:** [Scanning SAST/DAST, deploying WAF/Parameter Store, and generating vulnerability reports](1.10-week10/)

* **Week 11:** [Fine-tuning WAF Rules, handling false positives, simulating DDoS attacks with K6, and finalizing project handover](1.11-week11/)

In summary, the 12-week journey not only deepened my foundation of AWS knowledge but also helped me refine my Security Architecture mindset and system operation capabilities in a real-world environment.
